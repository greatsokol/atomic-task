# Тестовое задание
## Описание:
  
Разработать приложение, имеющее в своем составе Master-Detail таблицы. 

На форме имеются 2 таблицы.

  Master - список документов с полями:
  - номер
  - дата
  - сумма
  - примечание

  Detail - список спецификаций с полями:
  - наименование
  - сумма

  Классическая связка один ко многим. Требуется реализовать добавление/редактирование/удаление документа и его спецификации (набора позиций).
  При этом сумма документа не редактируется, а пересчитывается на основе спецификаций.
  
### Дополнительно:
  Реализовать проверку уникальности значения номера документа и/или спецификации на уровне
  транзакции, с выдачей сообщения пользователю и с логированием ошибки в отдельную таблицу БД (в
  отдельной транзакции).

## Реализация

### Требования
  - JDK 17
  - Maven
  - PostgreSQL
  - Доступ к репозиториям для загрузки библиотек

### Предварительная настройка
- создайте в PostgreSQL базу данных, 
- укажите адрес СУБД и название БД в файле `src/main/resources/application.yaml` в настройке `datasource.url`, 
  - например `jdbc:postgresql://localhost:5432/<название БД>?currentSchema=<ваша схема, если отличается от public>`.
- укажите логин и пароль пользователя СУБД в настройках `datasource.user` и `datasource.password` соответственно

### Запуск
- в каталоге `back` выполните команду:
  - `mvn springboot:run`
- в каталоге front выполните команды:
  - `npm install`
  - `npm start`
